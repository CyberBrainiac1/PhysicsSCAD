# Interactive Learning Guide

PhysicsSCAD is now meant to be used like a **hands-on physics workbook**, not a model gallery.

## The Core Loop: Predict -> Manipulate -> Explain
1. **Predict**: Before touching the model, write what you think will happen.
2. **Manipulate**: Move parts, swap tokens, change geometry, and observe.
3. **Explain**: Defend what happened with physics language (forces, energy, momentum, torque, constraints).

If your prediction fails, that is a win: it reveals exactly what to fix in your mental model.

## Draw Before Touching
For vector, force, and motion models:
- Sketch a quick diagram first.
- Label known directions/signs.
- Mark what is changing and what is constrained.

Then use the physical model to test the sketch.

## Change One Thing At A Time
Most misconception traps come from changing too much at once.

Use this rule for every activity:
- Keep all but one variable fixed (angle, mass, radius, lever arm, friction condition, COM height, etc.).
- Compare outcomes directly.
- State exactly which change caused the observed difference.

## Use the Model, Then Solve a Problem
After each physical activity:
1. Solve one matching AP Physics 1 style problem.
2. Solve one “what if?” variant.
3. Write one sentence connecting the equation to the model geometry.

This is how intuition becomes transferable problem-solving skill.

## Student Mode vs Mentor Mode

### Student Mode (Solo)
- Use the folder README Quick Start.
- Run 3 activities, then 1 challenge prompt.
- Keep a short lab notebook:
  - Prediction
  - Observation
  - Explanation
  - Mistake fixed

### Mentor Mode (Pair/Coach)
- Student predicts first, mentor stays silent.
- Mentor asks only constraint questions:
  - “What must stay equal?”
  - “What changed, exactly?”
  - “Which force/torque term is dominant?”
- End with student teaching back the model in 60 seconds.

## AP Physics 1 Workflow
1. Vectors/components and graph interpretation.
2. Newton’s laws + free-body diagrams + friction.
3. Energy, momentum, impulse.
4. Torque, rotation, COM, rolling.
5. Oscillations and fluids (concept-focused reinforcement).

Use the models to build **diagram discipline** and **direction/sign discipline**.

## F=ma Preparation Workflow
Use challenge prompts for:
- model selection (force vs energy vs momentum vs torque)
- comparing limiting cases
- checking consistency when assumptions change
- avoiding equation-first reflexes

Goal: choose the right framework quickly before calculating.

## Robotics Intuition Workflow
For each robotics model, answer:
1. What changes if payload moves outward/upward?
2. What tradeoff is being made (speed vs torque, stability vs reach, energy storage vs response)?
3. What failure mode does this predict on a real FTC-style robot?

## Avoid Passive Browsing
Passive: “Looks cool.”
Active: “I predicted X, observed Y, and corrected assumption Z.”

If you cannot write the corrected assumption, rerun the activity with a simpler setup.

## Suggested Weekly Cadence (Spring-Summer)
- **3 sessions/week**, 30–60 min each.
- Session structure:
  - 10 min prediction + setup
  - 15–30 min model work
  - 10–20 min problem transfer
- End every week with one mixed challenge from `CHALLENGE_SETS.md`.

## Minimal Materials Checklist
- Printed model set for current phase
- Dry-erase marker / sticky labels
- Small ruler, cheap scale (optional), phone timer
- Notebook dedicated to prediction/observation/explanation logs
