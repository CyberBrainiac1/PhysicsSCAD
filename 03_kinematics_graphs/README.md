# Kinematics Graph Tile Set

Model: `kinematics_graph_tile_set.scad`

## What This Is
Modular graph tiles for building x-t, v-t, and a-t stories physically.

## What It Teaches
Slope meaning, piecewise motion, and translation between graph families.

## Why This Matters
Students often memorize graph shapes without physical meaning; this kit forces explicit mapping.

## How To Print
Print tile families in contrasting colors. Keep pegs/snaps slightly oversized only if printer tolerances require.

## Quick Start
Build a two-segment v-t graph, then sketch matching a-t from slope signs.

## How to Learn With This
### Student mode
- Build graph first from words (“speed up then cruise”), then verify by slope logic.

### Mentor mode
- Mentor asks for narration: “What is the object doing right now?” at each segment.

### Hands-on learning tasks
1. Build constant positive velocity and identify x-t shape.
2. Build v-t with increasing slope; infer constant positive acceleration.
3. Build piecewise stop-start motion and mark turning points.
4. Make two motions with same final position but different v-t structure.
5. Build a-t first, then generate one consistent v-t.

## Things to Predict Before Using
- If v-t crosses zero, what must happen to x-t slope?
- Can v=0 while a!=0 at an instant?
- Which graph changes first when acceleration flips sign?

## Challenge Prompts
- Build motion where position increases overall, but velocity is negative briefly.
- Make two different v-t graphs with equal displacement interval.
- Create a physically valid “pause then reverse” story with all three graphs.

## Explanation Prompts
- Why is slope the bridge between the graph families?
- Why can same displacement come from different velocity histories?
- Why does flat x-t imply zero velocity, not necessarily zero acceleration forever?

## Common Mistakes / Misconceptions
- Confusing graph height with slope.
- Treating negative velocity as “slowing down” automatically.
- Assuming zero velocity means zero acceleration.
