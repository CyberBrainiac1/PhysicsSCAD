# Momentum / Collision Token Set

Model: `momentum_collision_token_set.scad`

## What This Is
Mass-labeled tokens and lane/board guides for 1D and 2D collision setups.

## What It Teaches
Momentum as a vector, impulse intuition, and mass-speed tradeoffs.

## Why This Matters
Students often overfocus on speed and underweight mass/vector direction.

## How To Print
Print multiple token masses with clear labels. Print lane board with high-contrast direction markers.

## Quick Start
Choose two tokens with different masses, assign equal speeds, and predict total momentum comparison.

## How to Learn With This
### Student mode
- Run one 1D and one 2D setup each session.

### Mentor mode
- Ask for momentum vectors before discussing outcomes.

### Hands-on learning tasks
1. Compare equal-speed, different-mass tokens.
2. Build head-on equal-mass collision narrative.
3. Build inelastic “stick together” scenario with token pairing.
4. Use velocity arrow slots to compare before/after vector sums.
5. Create 2D oblique-collision setup and discuss component-wise momentum.

## Things to Predict Before Using
- Can a slower heavier token carry more momentum than a faster lighter token?
- What stays conserved in isolated collision systems?
- In which collisions is kinetic energy not conserved?

## Challenge Prompts
- Build two distinct initial setups with same total momentum vector.
- Construct a collision where one object stops but system momentum is still conserved.

## Explanation Prompts
- Why is momentum a vector while kinetic energy is scalar?
- Why does conservation apply to system totals, not each object individually?

## Common Mistakes / Misconceptions
- “Bigger object always has bigger momentum.”
- Confusing force with momentum.
- Assuming kinetic energy is always conserved in collisions.

Momentum conservation, impulse, collision classification.
